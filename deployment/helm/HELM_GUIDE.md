# Q-Shield Helm Deployment Guide

## Quick Start

```bash
# Add Helm repositories
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo add grafana https://grafana.github.io/helm-charts
helm repo add elastic https://helm.elastic.co
helm repo update

# Navigate to deployment directory
cd deployment/helm

# Install (development)
helm install qshield ./qshield -n qshield --create-namespace

# Install (production)
helm install qshield ./qshield \
  -f values-production.yaml \
  -n qshield \
  --create-namespace
```

## Prerequisites

- Kubernetes 1.24+
- Helm 3.x
- kubectl configured to your cluster
- PersistentVolume provisioner (StorageClass)

## Values Configuration

### Development Defaults

Default values in `qshield/values.yaml`:
- 3 API replicas with auto-scaling 3-10
- PostgreSQL 16 with 50GB storage
- Redis 7 with AOF persistence
- All services enabled (Prometheus, Grafana, Elasticsearch, Kibana)

### Production Overrides

Use `values-production.yaml` for production deployments:
- 3-20 auto-scaling replicas (higher max)
- PostgreSQL 16 with 100GB storage + replication
- Redis 7 cluster mode, 3 replicas
- 90-day metric retention
- Fast SSD storage classes
- TLS/SSL configured via cert-manager
- Rate limiting on ingress

### Custom Values

Create your own `values-custom.yaml`:

```yaml
# Override specific values
replicaCount: 5

resources:
  limits:
    cpu: 2000m
    memory: 4Gi

postgresql:
  auth:
    password: "your-secure-password"
  primary:
    persistence:
      size: 200Gi

# Your custom settings...
```

## Installation

### Development Installation

```bash
# Minimal setup (local testing)
helm install qshield ./qshield \
  -n qshield \
  --create-namespace \
  --set postgresql.auth.password="devpass" \
  --set redis.auth.password="devpass"

# Wait for pods
kubectl get pods -n qshield -w
```

### Production Installation

```bash
# Production deployment with full monitoring
helm install qshield ./qshield \
  -f values-production.yaml \
  -n qshield \
  --create-namespace

# Verify installation
helm status qshield -n qshield

# View resources
kubectl get all -n qshield
```

### Staging Installation

```bash
# Create staging-specific values
helm install qshield-staging ./qshield \
  -f values-staging.yaml \
  -n qshield-staging \
  --create-namespace
```

## Upgrade

```bash
# Upgrade to new version
helm upgrade qshield ./qshield -f values-production.yaml -n qshield

# Rollback if needed
helm rollback qshield 1 -n qshield
```

## Configuration

### Environment Variables

Set via `env` section in values:

```yaml
env:
  ENVIRONMENT: production
  LOG_LEVEL: WARNING
  DEBUG: "false"
  ALLOWED_HOSTS: '["api.yourdomain.com"]'
  CORS_ORIGINS: '["https://yourdomain.com"]'
```

### Secrets

Secrets are autogenerated from `values.yaml`:
- Database password (PostgreSQL)
- Redis password
- SECRET_KEY (random 32 chars)
- OAuth credentials (Google, GitHub, Microsoft)

To use existing secrets:

```bash
# Create secret first
kubectl create secret generic qshield-secrets \
  --from-literal=DATABASE_PASSWORD=mypass \
  -n qshield

# Reference in values
postgresql:
  auth:
    existingSecret: qshield-secrets
```

### Persistence

Configure storage:

```yaml
persistence:
  enabled: true
  storageClassName: fast-ssd  # Must exist in your cluster
  size: 100Gi
  mountPath: /var/lib/qshield
```

### Resource Limits

Production settings:

```yaml
resources:
  limits:
    cpu: 1000m
    memory: 2Gi
  requests:
    cpu: 500m
    memory: 1Gi
```

### Autoscaling

Configure HPA:

```yaml
autoscaling:
  enabled: true
  minReplicas: 3
  maxReplicas: 20
  targetCPUUtilizationPercentage: 70
  targetMemoryUtilizationPercentage: 80
```

## Backup & Restore

### Backup Database

```bash
# Create backup
kubectl exec -it postgres-0 -n qshield -- \
  pg_dump -U qshield qshield > backup.sql
```

### Restore Database

```bash
# Restore from backup
kubectl exec -i postgres-0 -n qshield -- \
  psql -U qshield qshield < backup.sql
```

### Backup Persistent Data

```bash
# Use Velero or similar for cluster-wide backup
velero backup create qshield-backup -n qshield
```

## Scaling

### Horizontal Scaling (more pods)

```bash
# Via HPA (automatic)
kubectl autoscale deployment qshield \
  --min=3 --max=20 \
  -n qshield

# Manual scaling
kubectl scale deployment qshield \
  --replicas=5 \
  -n qshield
```

### Vertical Scaling (more resources)

```bash
# Update values
helm upgrade qshield ./qshield \
  --set resources.limits.cpu=2000m \
  --set resources.limits.memory=4Gi \
  -n qshield
```

## Common Issues

### Pods stuck in Pending

```bash
# Check events
kubectl describe pod <pod-name> -n qshield

# Common causes:
# - Insufficient CPU/memory resources
# - PersistentVolume not available
# - Image pull issues

# Check events
kubectl get events -n qshield
```

### Database connection failing

```bash
# Check PostgreSQL is running
kubectl get pod -l app.kubernetes.io/name=postgresql -n qshield

# Check logs
kubectl logs postgresql-0 -n qshield

# Test connection
kubectl exec -it qshield-0 -n qshield -- \
  psql -h postgresql -U qshield -d qshield -c "SELECT 1"
```

### Out of memory errors

```bash
# Check memory usage
kubectl top pod -n qshield

# Increase limits
helm upgrade qshield ./qshield \
  --set resources.limits.memory=4Gi \
  -n qshield
```

## Best Practices

1. **Always use specific versions** - Don't use 'latest' in production
2. **Use values files** - Separate dev/staging/prod values
3. **Enable monitoring** - ServiceMonitor for Prometheus
4. **Enable persistence** - Don't lose data on pod restart
5. **Set resource limits** - Prevent resource exhaustion
6. **Use ingress** - Proper routing and TLS
7. **Enable HPA** - Auto-scale based on metrics
8. **Regular backups** - Test restore procedures
9. **Review logs regularly** - Catch issues early
10. **Monitor metrics** - Use Prometheus + Grafana
